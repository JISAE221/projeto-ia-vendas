Modelo Snowflake

fVendas no centro. dProduto se liga a dCategoria. dVendedor tem self-join via IDGERENTE. Diferente do Star Schema — tem sub-dimensões.

Hierarquia de Vendedores

3 gerentes (Eduardo, Lilian, David). Cada um gerencia um grupo. Criar coluna calculada no Power Query ou DAX pra trazer o nome do gerente pelo IDGERENTE.

TOTAL_ITEM já calculado

A fVendas já tem TOTAL_ITEM, CUSTO_TOTAL e LUCRO_TOTAL por linha. Medidas DAX são somas/médias dessas colunas — sem precisar recalcular preço × quantidade.

⚠ Datas em 2102: alguns registros de fVendas têm IDTEMPO como Timestamp de 2102 — provável erro de importação. No Power Query, fazer join pelo campo data e filtrar ou corrigir esse range antes de modelar.
ℹ SCDs: dCliente, dFornecedor e dVendedor têm colunas INICIO e FIM — padrão de Slowly Changing Dimension tipo 2. Para este projeto, filtrar apenas registros com FIM = null (registro atual).
